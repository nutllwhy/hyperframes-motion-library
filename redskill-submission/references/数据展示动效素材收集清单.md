# 数据展示动效素材收集清单

> 收集日期：2026-06-20
> 用途：为视频动效系统补充可被 HTML 化的通用数据展示模板
> 现有模板：metric-pulse（数据脉冲）、stat-duel（双数据对比）、status-split（状态转折）、number-impact（极值刻度）

---

## 收集原则

1. **通用性强**：可适配不同数据类型，不限定具体行业场景
2. **HTML 可化**：核心动画可用 SVG + CSS + GSAP 实现，不依赖复杂 3D 或粒子系统
3. **视频友好**：1920×1080 横屏，字体与图形在视频压缩后仍可读
4. **参数化**：数据、文案、颜色、时长可替换，动画结构保留在模板内
5. **确定性动画**：不依赖随机时间，可任意帧寻址

---

## 一、图表增长类（Chart Growth）

### 1. bar-chart-grow | 柱状图增长

- **分类**：数据可视化
- **标签**：柱状图、增长、对比、多系列
- **场景**：年度业绩对比、多产品销量、地区数据分布
- **透明背景**：否（深色背景）
- **时长**：5–7 秒
- **视觉描述**：
  - 底部 X 轴标签，左侧 Y 轴刻度（可选）
  - 3–7 根垂直柱形，从底部以弹性缓动向上增长
  - 柱形顶部显示数值标签，延迟 0.2s 后淡入
  - 柱形可选不同颜色区分系列，或统一品牌色
  - 背景可带极细网格线，不喧宾夺主
- **动画拆解**：
  - 0.0s：背景网格线淡入（0.5s）
  - 0.3s：第一根柱形从 `scaleY: 0` 增长至目标高度（0.8s，elastic.out）
  - 0.5s：第二根柱形…以此类推，错开 0.15s
  - 柱形到达高度后，顶部数值标签从上方滑入 + 淡入（0.4s）
  - 最后 1s：整体稳定展示，可选微呼吸效果（scale 1.0→1.01 循环）
- **可变参数**：
  - `bars`：数组，每项含 `{ label, value, color }`
  - `maxValue`：Y 轴最大值（自动计算或手动指定）
  - `title`：图表标题
  - `subtitle`：副标题
  - `primaryColor`：柱形主色
  - `showGrid`：是否显示网格线
  - `showValues`：是否显示数值标签
- **HTML 实现要点**：
  - SVG `<rect>` 元素，用 `transform-origin: bottom` 做 scaleY 动画
  - GSAP `stagger` 控制错开时机
  - 数值标签用绝对定位 `div` 覆盖在柱形上方
- **参考来源**：After Effects 条形图模板、AECharts 默认动画、数据新闻视频常见样式

---

### 2. line-chart-draw | 折线绘制

- **分类**：数据可视化
- **标签**：折线图、趋势、时间序列、绘制动画
- **场景**：股价走势、用户增长趋势、温度变化、月度数据
- **透明背景**：否（深色背景）
- **时长**：6–8 秒
- **视觉描述**：
  - 底部 X 轴时间标签，左侧 Y 轴刻度
  - 单条或多条折线，从左侧逐点绘制到右侧
  - 折线下方可选填充渐隐面积（area chart）
  - 数据点用圆点标记，到达后放大弹出
  - 终点可带特殊标记（如最新数据高亮）
- **动画拆解**：
  - 0.0s：坐标轴与网格线淡入（0.6s）
  - 0.5s：折线从起点开始 `stroke-dashoffset` 绘制（2–3s，power2.inOut）
  - 绘制过程中，每个数据点圆点依次放大出现（scale 0→1，back.out）
  - 折线绘制完成后，下方填充区域从底部向上淡入（0.8s）
  - 最后 1.5s：终点数据标签弹出，带数值和说明
- **可变参数**：
  - `points`：数组，每项 `{ x, y, label }`
  - `lineColor`：折线颜色
  - `fillColor`：下方填充渐变（如 `rgba(255,90,54,0.2)` 到 `rgba(255,90,54,0)`）
  - `strokeWidth`：线宽（2–4px）
  - `showDots`：是否显示数据点
  - `dotRadius`：数据点半径
  - `title`：图表标题
- **HTML 实现要点**：
  - SVG `<path>` + `stroke-dasharray` / `stroke-dashoffset` 动画
  - GSAP `drawSVG` 插件或手动计算 path length
  - 数据点用 `<circle>`，错开动画时机
- **参考来源**：Envato Line Graph 模板、Flourish 折线动画、数据新闻趋势图

---

### 3. pie-chart-expand | 饼图/环形展开

- **分类**：数据可视化
- **标签**：饼图、环形图、占比、构成
- **场景**：市场份额、预算分配、用户构成、投票结果
- **透明背景**：可选（深色背景或透明）
- **时长**：5–6 秒
- **视觉描述**：
  - 中心环形或实心饼图，扇区从 0° 逐片展开
  - 每片扇区带颜色区分，色值需协调
  - 中心可显示总计或核心指标
  - 扇区外侧带引线标签，显示类别名和百分比
  - 当前高亮扇区可带微光晕或偏移效果
- **动画拆解**：
  - 0.0s：中心标题或总计数字淡入（0.5s）
  - 0.3s：第一扇区从角度 0 展开到目标角度（0.6s，power2.out）
  - 后续扇区依次错开 0.15s 展开
  - 所有扇区展开后，引线标签从扇区边缘向外滑入（0.5s）
  - 最后 1s：当前最大占比扇区轻微放大（scale 1.05）并加光晕
- **可变参数**：
  - `slices`：数组，每项 `{ label, value, color }`
  - `innerRadius`：内径（0 为饼图，>0 为环形图）
  - `outerRadius`：外径
  - `centerText`：中心文字（如总计）
  - `centerValue`：中心数值
  - `showLabels`：是否显示引线标签
  - `highlightLargest`：是否高亮最大占比
- **HTML 实现要点**：
  - SVG `<path>` 用 `arc` 命令绘制扇形
  - 通过 `d` 属性插值实现展开动画，或 `transform: rotate` 配合 `clip-path`
  - 引线用 SVG `<line>` + `<text>` 或绝对定位 `div`
- **参考来源**：Infographic Pie Chart 模板、Canva 数据图表、企业年报视频

---

### 4. area-chart-fill | 面积图填充

- **分类**：数据可视化
- **标签**：面积图、趋势、填充、堆叠
- **场景**：累计增长、多指标堆叠、市场份额变化
- **透明背景**：否（深色背景）
- **时长**：6–7 秒
- **视觉描述**：
  - 类似折线图，但折线下方有渐变填充区域
  - 多系列时可堆叠，每层颜色不同且透明度递减
  - 数据点标记可选
  - 整体呈现"数据流"的流动感
- **动画拆解**：
  - 0.0s：坐标轴淡入
  - 0.5s：折线从左侧绘制到右侧（2s）
  - 折线绘制同时，下方填充区域从底部向上展开（clip-path 或 mask 动画）
  - 多系列时，系列间错开 0.3s
  - 最后 1s：数据点依次弹出
- **可变参数**：
  - `series`：数组，每项 `{ name, points[], color }`
  - `stacked`：是否堆叠
  - `fillOpacity`：填充透明度（0.1–0.4）
  - `smooth`：是否平滑曲线（贝塞尔）
- **HTML 实现要点**：
  - SVG `<path>` 绘制区域，用 `clip-path` 动画实现从下往上展开
  - 渐变填充用 SVG `<linearGradient>`
- **参考来源**：数据可视化大屏、年报视频趋势图、Flourish area chart

---

### 5. bar-chart-race | 条形图竞赛

- **分类**：数据可视化
- **标签**：条形图、竞赛、排序、时间序列、动态排名
- **场景**：历年公司排名变化、城市 GDP 竞赛、产品销量排行变化
- **透明背景**：否（深色背景）
- **时长**：8–10 秒（或更长，支持时间轴驱动）
- **视觉描述**：
  - 水平条形，左侧是类别标签，右侧是数值
  - 条形随时间排序变化，条形长度动态增长/缩短
  - 当前时间标签（如年份）在角落跳动变化
  - 顶部 N 名（如 Top 5）可见，其余淡出
- **动画拆解**：
  - 时间轴驱动：每帧更新条形长度和排序
  - 条形位置变化用 GSAP `y` 属性动画（0.3s，power2.inOut）
  - 条形长度变化用 `scaleX` 或 `width` 动画
  - 数值标签同步变化（CountUp 效果）
  - 时间标签跳动时带缩放脉冲（scale 1.2→1.0）
- **可变参数**：
  - `timelineData`：对象，键为时间标签，值为 `{ label, value }` 数组
  - `topN`：显示前几名（如 5）
  - `timeLabel`：当前时间标签位置（左上/右上）
  - `barHeight`：每根条形高度
  - `colorMap`：类别到颜色的映射
- **HTML 实现要点**：
  - 用 GSAP `gsap.to()` 驱动条形 `y` 和 `width` 属性
  - 排序变化时重新计算位置，GSAP 处理过渡动画
  - 时间标签用 `requestAnimationFrame` 或 GSAP `timeline` 控制
- **参考来源**：Flourish bar chart race、YouTube 数据可视化频道（如 "The Rankings"）
- **难度**：高（排序动画和状态管理较复杂）

## 二、数字计数类（Number Animation）

### 6. number-counter | 数字计数器

- **分类**：数据可视化
- **标签**：计数器、数字、滚动、KPI、大数字
- **场景**：粉丝增长、销售额突破、累计用户数、里程碑
- **透明背景**：可选（纯色或透明）
- **时长**：4–5 秒
- **视觉描述**：
  - 巨型数字（占画面 30–50% 宽度），从 0 滚动到目标值
  - 数字等宽，滚动时有"老虎机"效果
  - 数字前/后可带单位（如 "¥"、"万"、"+"）
  - 下方或上方带标题说明
  - 可选进度环或背景装饰线
- **动画拆解**：
  - 0.0s：标题文字淡入（0.5s）
  - 0.3s：数字开始滚动，从 0 到目标值（3s，power2.out）
  - 滚动过程中，数字位依次减速，最终位最后一个到达
  - 数字停止时，整体带轻微弹跳（scale 1.02→1.0，0.2s）
  - 单位标签从右侧滑入（0.4s）
  - 可选：背景进度环同步旋转到目标角度
- **可变参数**：
  - `targetValue`：目标数值（整数或小数）
  - `prefix`：前缀（如 "¥"、"+"）
  - `suffix`：后缀（如 "万"、"%"）
  - `duration`：滚动时长（秒）
  - `decimalPlaces`：小数位数
  - `separator`：千位分隔符（如 ","）
  - `title`：标题文字
  - `showProgressRing`：是否显示进度环
- **HTML 实现要点**：
  - 用 `CountUp.js` 或 GSAP `onUpdate` 手动插值
  - 等宽数字用 `font-variant-numeric: tabular-nums` 或等宽字体
  - 进度环用 SVG `<circle>` + `stroke-dashoffset`
- **参考来源**：After Effects Counter 模板、网站数据展示、发布会 Keynote

---

### 7. multi-metric-grid | 多指标网格

- **分类**：数据可视化
- **标签**：多指标、网格、KPI、仪表盘、卡片
- **场景**：运营日报、季度总结、企业数据看板、实时监控
- **透明背景**：可选（深色背景或透明）
- **时长**：6–8 秒
- **视觉描述**：
  - 2×2、2×3 或 3×3 网格布局，每个格子是一个 KPI 卡片
  - 每张卡片包含：指标名称、大数字、变化趋势（↑/↓ + 百分比）、迷你图
  - 卡片逐个入场，带错开动画
  - 整体像数据仪表盘截图，但带动态入场
- **动画拆解**：
  - 0.0s：第一行卡片从下方滑入（0.6s，power3.out）
  - 0.2s：第二行卡片…以此类推，行间错开 0.2s
  - 每张卡片内部：标题先淡入（0.3s），然后数字滚动（1s），最后趋势图标弹出（0.3s）
  - 迷你图（sparkline）在卡片内同步绘制
  - 最后 1s：整体稳定，可选某张卡片脉冲高亮
- **可变参数**：
  - `metrics`：数组，每项 `{ title, value, change, trend, sparklinePoints[], color }`
  - `columns`：每行列数（2–4）
  - `cardColor`：卡片背景色（或统一背景）
  - `accentColor`：趋势上升/下降颜色
  - `showSparkline`：是否显示迷你图
- **HTML 实现要点**：
  - CSS Grid 布局，响应式列数
  - GSAP `stagger` 控制卡片入场
  - 迷你图用 SVG `<polyline>` 或 `<path>` 简单绘制
- **参考来源**：Dashboard UI 设计、数据大屏、企业管理视频

## 三、进度与状态类（Progress & Status）

### 8. progress-ring | 环形进度条

- **分类**：数据可视化
- **标签**：进度条、环形、百分比、完成度、仪表盘
- **场景**：任务完成率、目标达成度、投票进度、健康指标
- **透明背景**：可选（纯色或透明）
- **时长**：4–5 秒
- **视觉描述**：
  - 中心大数字显示百分比（如 87%）
  - 环形轨道（浅色/暗色）+ 进度弧（品牌色）
  - 进度弧从 0° 旋转到目标角度
  - 中心数字同步从 0 滚动到目标值
  - 可选标题和说明文字在下方或上方
- **动画拆解**：
  - 0.0s：环形轨道淡入（0.5s）
  - 0.3s：进度弧开始绘制（2.5s，power2.out）
  - 进度弧同时，中心数字从 0 滚动到目标值（同步时长）
  - 百分比符号在数字停止后从右侧滑入（0.3s）
  - 最后 1s：完成时，进度弧末端带光点闪烁（可选）
- **可变参数**：
  - `progress`：进度值（0–100）
  - `trackColor`：轨道颜色
  - `fillColor`：进度弧颜色
  - `strokeWidth`：环粗细（8–20px）
  - `title`：标题
  - `subtitle`：说明文字
  - `showPercent`：是否显示百分比符号
  - `startAngle`：起始角度（默认 -90°，即顶部）
- **HTML 实现要点**：
  - SVG `<circle>` + `stroke-dasharray` + `stroke-dashoffset`
  - 计算 circumference = 2 × π × r
  - 用 `transform: rotate(-90deg)` 让起点在顶部
- **参考来源**：加载动画、健身 App 数据展示、游戏 HUD

---

### 9. gauge-meter | 仪表盘

- **分类**：数据可视化
- **标签**：仪表盘、速度计、半圆、指标、危险/安全区间
- **场景**：满意度评分、风险等级、性能指标、健康状态
- **透明背景**：可选（纯色或透明）
- **时长**：4–6 秒
- **视觉描述**：
  - 半圆形仪表盘（180° 或 240°），底部平齐
  - 刻度线均匀分布，可选颜色分区（绿/黄/红）
  - 指针从 0 位置旋转到目标角度
  - 中心或下方显示数值
  - 可选当前状态标签（如 "优秀"、"危险"）
- **动画拆解**：
  - 0.0s：仪表盘底板和刻度淡入（0.6s）
  - 0.5s：颜色分区依次点亮（0.4s，stagger）
  - 1.0s：指针从起始角度旋转到目标角度（2s，elastic.out 或 back.out）
  - 指针旋转同时，中心数字滚动到目标值
  - 指针停止后，状态标签从下方弹出（0.5s，back.out）
  - 最后 1s：指针轻微抖动（模拟真实仪表）后稳定
- **可变参数**：
  - `value`：当前数值
  - `min` / `max`：最小/最大值范围
  - `zones`：颜色分区，每项 `{ from, to, color, label }`
  - `needleColor`：指针颜色
  - `title`：标题
  - `statusLabel`：状态标签（如 "正常"）
- **HTML 实现要点**：
  - SVG 绘制半圆弧和刻度线
  - 指针用 `<line>` 或 `<polygon>`，`transform: rotate()` 动画
  - 刻度线用循环生成 `<line>` 或 `<path>`
- **参考来源**：汽车仪表盘、数据监控大屏、金融风控视频

## 四、列表与排名类（List & Ranking）

### 10. rank-list-reveal | 排行榜揭示

- **分类**：数据可视化
- **标签**：排行榜、列表、排名、Top N、揭示
- **场景**：Top 10 榜单、城市排名、产品排行、销售排名
- **透明背景**：可选（深色背景或透明）
- **时长**：6–10 秒（取决于条目数）
- **视觉描述**：
  - 垂直列表，每项包含：排名序号、名称、数值、进度条
  - 从第 1 名到第 N 名依次揭示，错开入场
  - 第 1 名可特殊高亮（更大字号、不同颜色、金色边框）
  - 每项右侧有数值进度条，宽度对应数值大小
- **动画拆解**：
  - 0.0s：标题 "TOP 10" 或榜单名称淡入（0.5s）
  - 0.5s：第 1 名从左侧滑入（0.6s，power3.out），同时进度条从 0 增长至目标宽度（0.8s）
  - 第 2 名延迟 0.2s 后入场…以此类推
  - 排名序号先弹出（scale 0→1.2→1），然后文字滑入
  - 最后 1s：整体稳定，第 1 名带微光晕或脉冲效果
- **可变参数**：
  - `items`：数组，每项 `{ rank, name, value, color }`
  - `maxValue`：用于计算进度条比例
  - `highlightFirst`：是否高亮第 1 名
  - `showProgressBar`：是否显示进度条
  - `itemCount`：显示条目数（3–10）
  - `title`：榜单标题
- **HTML 实现要点**：
  - CSS Flexbox 或 Grid 列表布局
  - GSAP `stagger` 控制列表项错开入场
  - 进度条用 `div` + `width` 动画
- **参考来源**：YouTube 榜单视频、音乐排行榜、年终盘点

---

### 11. timeline-horizontal | 水平时间轴

- **分类**：数据可视化
- **标签**：时间轴、里程碑、历史、进程、时间线
- **场景**：公司发展历程、项目里程碑、产品迭代史、年度大事记
- **透明背景**：可选（纯色或透明）
- **时长**：6–8 秒
- **视觉描述**：
  - 水平主线贯穿画面，从左到右
  - 节点均匀分布在线上，每个节点带日期和事件标题
  - 节点圆点从左侧依次放大出现
  - 事件文字在节点上方或下方交替排列，避免拥挤
  - 当前节点可高亮，已完成节点带对勾或实心标记
- **动画拆解**：
  - 0.0s：水平主线从左侧 `scaleX: 0` 展开到全宽（1.5s，power2.inOut）
  - 0.5s：第一个节点圆点从中心放大（scale 0→1，back.out，0.4s）
  - 节点出现后，事件文字从上方/下方滑入（0.5s），错开 0.3s
  - 节点间可用细线连接，同步绘制
  - 最后 1s：当前/最新节点脉冲高亮
- **可变参数**：
  - `milestones`：数组，每项 `{ date, title, description, position }`
  - `lineColor`：时间轴主线颜色
  - `nodeColor`：节点颜色（默认品牌色）
  - `activeNodeColor`：当前节点高亮色
  - `alternateLabels`：文字是否交替上下排列
  - `title`：时间轴标题
- **HTML 实现要点**：
  - SVG `<line>` 或 `div` 做水平轴线
  - 节点用绝对定位的 `div` 圆点
  - GSAP `stagger` 控制节点和文字错开
- **参考来源**：企业宣传片、路演 PPT、历史回顾视频

- **参考来源**：操作流程视频、产品发布步骤、教程视频

## 五、特殊数据展示类（Specialized Data）

### 13. funnel-chart | 漏斗图

- **分类**：数据可视化
- **标签**：漏斗、转化、递减、层级、步骤
- **场景**：销售漏斗、用户转化、招聘流程、筛选过程
- **透明背景**：可选（纯色或透明）
- **时长**：5–7 秒
- **视觉描述**：
  - 倒三角形漏斗，分 3–5 层，每层宽度递减
  - 每层带标题和数值，可选转化率百分比
  - 整体从上方滑入，每层依次展开
  - 层间可用流动粒子或连接线表示"流动"
- **动画拆解**：
  - 0.0s：漏斗整体从上方滑入（0.8s，power3.out）
  - 0.5s：第一层宽度从 0 展开到目标（0.6s）
  - 随后每层依次向下展开，错开 0.2s
  - 每层展开后，数值标签从中心弹出（0.4s）
  - 层间转化率标签在两层都展开后淡入（0.3s）
  - 最后 1s：整体稳定，可选流动粒子效果
- **可变参数**：
  - `stages`：数组，每项 `{ name, value, color }`
  - `showConversionRate`：是否显示层间转化率
  - `flowEffect`：是否显示流动粒子
  - `title`：漏斗标题
- **HTML 实现要点**：
  - SVG `<path>` 或 `<polygon>` 绘制梯形层
  - 用 `clip-path` 或 `scaleX` 动画实现展开
- **参考来源**：销售漏斗视频、运营数据分析、SaaS 指标展示

### 14. heatmap-grid | 热力矩阵

- **分类**：数据可视化
- **标签**：热力图、矩阵、网格、颜色映射、密度
- **场景**：日历热力图、区域密度、时间分布、活跃度矩阵
- **透明背景**：否（深色背景）
- **时长**：5–7 秒
- **视觉描述**：
  - 网格布局（如 7×5 日历格、或 10×10 矩阵）
  - 每个格子颜色深浅表示数值高低，从品牌色渐变到深色
  - 格子从随机位置或从左上角开始依次点亮
  - 可带行列标签（如星期、月份）
- **动画拆解**：
  - 0.0s：网格边框和标签淡入（0.5s）
  - 0.5s：格子开始依次点亮，从颜色 A（低值）渐变到颜色 B（高值）
  - 点亮顺序可以是：从左到右、随机、或从中心向外扩散
  - 每个格子点亮时间 0.2s，错开 0.05s
  - 最后 1s：高亮格子带微光晕，图例说明淡入
- **可变参数**：
  - `gridData`：二维数组，每项为数值
  - `rows` / `columns`：行列数
  - `colorLow` / `colorHigh`：低值/高值颜色
  - `revealOrder`：揭示顺序（left-to-right / random / center-out）
  - `rowLabels` / `colLabels`：行列标签
  - `title`：标题
- **HTML 实现要点**：
  - CSS Grid 布局，每个格子是 `div`
  - GSAP `stagger` 控制点亮顺序，用 `from` 指定不同 `stagger` 模式
  - 颜色映射用 HSL 或 RGB 插值
- **参考来源**：GitHub 贡献日历、数据密度图、时间分布视频

### 15. radar-chart | 雷达图/蜘蛛图

- **分类**：数据可视化
- **标签**：雷达图、多维、对比、能力值、评估
- **场景**：能力评估、产品对比、多维度评分、用户画像
- **透明背景**：可选（纯色或透明）
- **时长**：5–7 秒
- **视觉描述**：
  - 多边形网格（通常为 5–8 轴），从中心向外辐射
  - 每条轴带标签，轴上有刻度线
  - 数据区域用多边形填充，从中心向外展开
  - 多系列时可叠加对比（用不同颜色填充，透明度递减）
- **动画拆解**：
  - 0.0s：网格线和轴标签淡入（0.6s）
  - 0.8s：数据多边形从中心逐点展开（1.5s，power2.out）
  - 每个顶点依次到达目标位置，错开 0.1s
  - 顶点到达后带圆点标记放大出现
  - 多系列时，系列间错开 0.5s
  - 最后 1s：数据区域稳定，可选描边发光效果
- **可变参数**：
  - `axes`：数组，轴标签（如 ["速度", "力量", "技巧", "耐力", "智力"]）
  - `series`：数组，每项 `{ name, values[], color }`
  - `maxValue`：每轴最大值
  - `fillOpacity`：填充透明度
  - `showDots`：是否显示顶点圆点
  - `showGrid`：是否显示网格线
- **HTML 实现要点**：
  - SVG `<polygon>`，计算每个顶点的极坐标位置
  - 用 `points` 属性动画实现展开，或从中心 `scale` 放大
  - 顶点圆点用 `<circle>`
- **参考来源**：游戏角色属性、产品评测、能力模型视频

### 16. sparkline-stat | 迷你统计条

- **分类**：透明叠加
- **标签**：迷你图、统计、趋势、底部叠加、字幕条
- **场景**：纪录片数据标注、新闻画面叠加、短视频底部数据

### 17. comparison-bars | 对比条形组

- **分类**：透明叠加

- **场景**：产品对比、前后对比、两组数据对比、实验结果

- **时长**：5–6 秒
- **视觉描述**：
  - 2–3 组水平对比条形，每组两条（A vs B）
  - 条形上方或左侧有类别标签
  - 数值在条形末端显示
  - 两组用不同颜色区分（如品牌色 vs 对比色）

  - 每组用 Flexbox 布局，条形用 `div` + `width` 动画

### 18. live-ticker-strip | 滚动数据条

- **分类**：透明叠加
- **标签**：滚动、数据条、实时、跑马灯、底部信息
- **场景**：新闻底部滚动条、实时数据、股价信息、赛事数据
- **透明背景**：是（带透明通道）
- **时长**：循环（可设定单次时长）
- **视觉描述**：
  - 画面底部横向条带，文字从右向左无限滚动
  - 每条数据包含：标签 + 数值 + 变化箭头
  - 数据项之间有分隔符
  - 可选上下边框线或背景色块
- **动画拆解**：
  - 0.0s：条带从底部滑入（0.5s）
  - 0.5s：数据开始从右向左滚动（linear，无限循环或固定时长）

  - 最后 0.5s：条带向下滑出退场

  - `showBorder`：是否显示上下边框

- **HTML 实现要点**：
  - CSS 动画 `translateX` 或 GSAP `x` 动画
  - 需要复制一份数据实现无缝循环
  - 使用 `display: flex` + `white-space: nowrap`
- **参考来源**：新闻频道、体育赛事直播、股市直播画面

## 八、视觉风格建议

基于现有模板（metric-pulse 等）的视觉规范，新增数据展示模板建议统一以下风格：

- **背景**：`#12100E`（深棕黑）或透明（叠加类）
- **前景文字**：`#F4EFE7`（暖白）
- **强调色**：`#FF5A36`（品牌橙）
- **辅助色**：`#B8ADA2`（ muted 暖灰）、`#D8CEC4`（亮 muted）
- **面板/卡片**：`#211C18`（深棕），圆角 16–24px
- **线条/网格**：`#3A3029`（细线），`#5A4840`（强调线）
- **字体**：`IBM Plex Mono`（等宽，数据感），中文回退系统字体
- **动效语气**：精确、机械、编辑感，避免弹性过度（除非刻意）
- **禁用**：蓝紫渐变、霓虹发光、网页卡片阴影、居中悬浮布局、饼图（除 pie-chart-expand 外）、复杂 3D
